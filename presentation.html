1דף הבורסה באפליקציה

דף תיק המניות מציג את כלל נירות הערך של המשפחה — מניות אמריקאיות (USD) ומניות/קרנות ישראליות (ILS).

לכל ניר מוצגים: שער אחרון, שינוי יומי, רווח/הפסד כולל, וכמות. הנתונים מגיעים מ-yfinance לשוק האמריקאי ומ-Bizportal לשוק הישראלי.

המניות שמצויות בתיק הן הקלט שהסוכן מקבל בכל הרצה — הוא בודק אותן ומתריע בהתאם.

2תפקיד הסוכן וכלים

הסוכן הוא אנליסט פיננסי אוטונומי שרץ אוטומטית כל יום. לשם כך יש לו 5 כלים:

get_us_stock_data(ticker)
שער עדכני ממניה אמריקאית — yfinance
get_il_stock_data(ticker)
שער קרן/מניה ישראלית — Bizportal (קוד מספרי TASE)
scan_guru_portfolio(guru)
סריקת 13F מ-Dataroma — פוזיציות חדשות וחיסולים של Buffett, Ackman, Burry
search_financial_news(ticker)
עד 3 כותרות מ-7 הימים האחרונים בלבד, עם תאריך ולינק — נקרא אוטומטית כשמניה זזה ≥5%
send_telegram_alert(message)
שליחת התראה ל-Telegram. אם לא מוגדר — הסוכן מסכם בטקסט וה-router שולח מייל

3לופ הסוכן — ReAct עם LangGraph

הסוכן בנוי על LangGraph ReAct — ה-LLM חושב ופועל בלולאה עד שמגיע לתשובה. הוא מחליט לבד איזה tool לקרוא ובאיזה סדר.

LLMGemini 2.5
🧠THINKמה לעשות?
⚡ACTקורא ל-tool
👁OBSERVEתוצאת הכלי
🧠 THINK"AAPL זינק +6% — אבדוק למה"
⚡ ACTsearch_financial_news("AAPL")
👁 OBSERVE"Apple AI partnership with OpenAI…"
🧠 THINK"משמעותי — שולח התראה"
⚡ ACTsend_telegram_alert("📊 PRICE ALERT…")
✅ DONEאין עוד פעולות — הלולאה נעצרת

כל סיבוב = צעד בלולאה: הסוכן חושב → פועל → מתבונן בתוצאה, עד שאין עוד מה לעשות. 

# LLM — Gemini 2.5 Flash, temperature=0 (דטרמיניסטי)
llm = ChatGoogleGenerativeAI(model="gemini-2.5-flash", temperature=0)

# שורה אחת שמרכיבה את כל הלופ
return create_react_agent(
    model=llm,
    tools=AGENT_TOOLS,    # 5 כלים
    prompt=SYSTEM_PROMPT,  # חוקי SENSE → THINK → ACT
)

🔁
תקרת לולאה
recursion_limit=50 על ה-ReAct. LLM שנכנס ללולאה נעצר ב-GraphRecursionError במקום קריאות Gemini אינסופיות.
🛡️
ולידציית קלט
לפני yfinance / הסקרייפר — בדיקת תקינות ticker: US דורש אות, IL דורש 4–9 ספרות. קלט פסול מוחזר כשגיאה ברורה שהסוכן מתקן.
✂️
הגבלת פלט
התראת טלגרם ריקה נדחית; הודעה מעל 4096 תווים נחתכת — שליחה מקוצרת במקום כשל 400 שקט.
🎯
בדיקת הזיה
PRICE ALERT שנשלח על ניר שלא בתיק מסומן ב-warning — מזהה אם הסוכן "המציא" התראה.
🎲
דטרמיניזם
temperature=0 — ה-LLM בוחר תמיד את התשובה הסבירה ביותר. אותו קלט → אותה התנהגות, בלי "יצירתיות" בהתראות פיננסיות.
⏱️
תקרת זמן
כל הרצה עטופה ב-timeout 180s. סוכן תקוע (סקרייפר/API איטי) נקטע ולא תוקע את ה-cron או צובר עלות.
🗂️
dedup התראות
מפתחות גורו:טיקר:רבעון נשמרים ב-Firestore. 13F רבעוני לא יתריע כל יום מחדש — התראה אחת לכל אירוע.
📊
תקציר ground-truth
מה הסוכן באמת עשה נקרא מהיסטוריית קריאות-הכלים, לא מהטקסט החופשי של ה-LLM — מקור אמת אובייקטיבי שעליו בנויות הבדיקות.
